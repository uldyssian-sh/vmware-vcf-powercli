# Contributing to VMware VCF PowerCLI

## Code of Conduct

This project adheres to enterprise standards and professional conduct.

## How to Contribute

### Reporting Issues

1. Check existing issues first
2. Use the issue template
3. Provide detailed reproduction steps
4. Include environment information

### Pull Requests

1. Fork the repository
2. Create a feature branch
3. Follow coding standards
4. Add tests for new functionality
5. Update documentation
6. Submit pull request

## Coding Standards

### PowerShell Style Guide

- Use approved verbs only
- Follow PascalCase for functions
- Use full parameter names
- Include comment-based help
- Implement proper error handling

### Security Requirements

- No hardcoded credentials
- Input validation required
- Use secure credential storage
- Follow principle of least privilege

### Testing

- All scripts must pass PSScriptAnalyzer
- Include Pester tests for functions
- Test on supported platforms

1. Automated checks must pass
2. Code review by maintainers
3. Security scan approval
